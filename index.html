프로젝트 소개

기존 데이팅 앱의 낮은 신뢰도와 결혼정보회사의 높은 진입 장벽을 해결하기 위해, 신뢰할 수 있는 중개인을 중심으로 한 폐쇄형 인맥 풀과 검증된 이용자 간 매칭을 핵심으로 하는 모바일 앱과 관리자 웹을 3개월 MVP로 구축합니다.

핵심 요구사항 분석

  • 3개 사용자 역할(중개인/내부이용자/외부이용자)별 차별화된 가입 절차·탐색 범위·매칭 플로우 설계
  • 사진 비공개 원칙의 프로필 관리와 가치관·직업 등 상세 프로필 데이터 구조화 (최종 매칭 시 사진 공개 트리거)
  • 일일 추천: 매일 소수 이성 프로필 추천, '관심' 표현 → 관심함 관리
  • 라운지: 설문·심리테스트 콘텐츠, 신뢰 등급 높은 중개인 주최 단체 미팅
  • 이용자 간 채팅 차단: 중개인-이용자 1:1 채팅만 허용, 중개인 주도 매칭 제안
  • 중개인 풀 관리: 본인 풀 내 매칭, 다른 중개인 풀과의 교차 매칭, 외부 이용자의 풀 가입 신청
  • 매칭 진행도 단계별 상태관리: 제안·수락·거절·결제·성사 상태 머신 설계
  • 이중 결제 채널: 코인(재화) 기반 인앱결제(iOS StoreKit / Google Play Billing) + 웹 PG(포트원) 동시 지원
  • 중개인 수수료 정산: 매칭 성사 시 중개인 수수료 계산·정산 내역 제공, 관리자 정산 승인
  • 관리자 시스템: 대시보드, 회원·중개인 승인 관리, 매칭 모니터링, 콘텐츠·정산 관리
  • 보안: 개인정보(연락처·사진) 최종 매칭 전까지 비공개, 채팅 E2E 수준 암호화, 개인정보 필드 암호화
  • 성능: 다중 사용자 실시간 채팅·매칭 요청 시 지연 없는 처리 (WebSocket + Redis Pub/Sub)
  • 디자인: '바닐라 브릿지' 레퍼런스 기반 웜톤·파스텔톤의 아늑하고 고급스러운 감성 UI

제안 포인트

1. 매칭 플랫폼 구축 경험에 기반한 안정적 딜리버리

디지털 명함·네트워킹 매칭 플랫폼(12 마이크로서비스, 63 API, 41 모바일 기능)을 3개월 내 런칭한 수행 경험, 그리고 일정·캘린더 소셜 앱 MVP, 복잡한 다중 역할(7 사용자 역할, 6계층 보안) B2B SaaS 구축 경험을 토대로, 본 프로젝트의 핵심인 중개인 중심 매칭 플로우와 관리자 승인 워크플로우를 안정적으로 설계·구현할 수 있습니다.

2. 3역할 × 3매칭 시나리오의 상태 머신 설계

중개인·내부이용자·외부이용자 × (풀 내/풀 교차/외부→중개인 신청) 매칭 시나리오는 결재·승인 워크플로우 엔진 구축 경험을 그대로 이식할 수 있는 구조입니다. 역할별 권한(RBAC)·매칭 상태(제안·수락·거절·결제·성사·취소)·정산 트리거를 상태 머신과 CASL 기반 권한 체계로 코드 수준에서 보장합니다.

3. 이중 결제 채널의 정책·검증 경험

웹 PG 결제(포트원/NICE KYC), 인앱결제(App Store/Google Play) 영수증 검증, 가상계좌 자동화, 복식부기 기반 정산 시스템을 다수 프로젝트에서 구축한 경험을 바탕으로, 코인 재화 충전·차감·환불·정산의 4대 흐름을 재화 원장(ledger) 구조로 설계합니다.

4. 프라이버시 중심 아키텍처

사진·연락처의 매칭 이전 비공개 요구는 개인정보 필드 레벨 암호화(AES-256)와 미디어 URL의 서명 기반 임시 접근 제어(Signed URL, TTL)로 구현합니다. 채팅은 AES-GCM 기반 암호화를 적용하고, 매칭 확정 이벤트에서만 연락처·사진 디크립션을 허용하는 정책 엔진을 적용합니다.

5. 크로스플랫폼(Flutter) 단일 코드베이스

Flutter 기반 다수 앱(ex. 소셜 캘린더, 헬스케어 관리, 게이미피케이션 앱)을 iOS/Android 동시 출시한 경험을 바탕으로, 3가지 역할 UI를 하나의 Flutter 코드베이스에서 역할별 라우팅·네비게이션으로 분기해 빠르게 개발합니다.

6. '바닐라 브릿지' 레퍼런스 기반 감성 UI

웜톤·파스텔톤의 아늑·고급스러운 무드보드를 기획 단계에서 Figma 디자인 시스템으로 구성합니다. 모바일 감성 앱 디자인(Flutter + Figma) 경험을 바탕으로 사용자가 신뢰를 느낄 수 있는 톤앤매너·여백·타이포그래피를 세팅합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다. 본 PRD는 클라이언트 제공 요구사항 정의서 기반으로 작성되었으며, 착수 후 상세 기획 단계에서 구체화됩니다.

1. 프로젝트 배경 및 목적

  • 기존 데이팅 앱의 낮은 신뢰도와 결혼정보회사의 높은 진입 장벽을 해결하는 신뢰 기반 인맥 중개 플랫폼 구축
  • 검증된 '중개인'을 중심으로 한 폐쇄형 인맥 풀 공유 및 교차 매칭으로 서비스 신뢰도 확보
  • 중개인에게 '더 쉽게, 더 많이, 정당한 보상'으로 중개 가능한 채널 제공
  • 3개월 내 MVP 런칭 후 시장 반응 기반 단계적 고도화

2. 사용자 정의

사용자 유형설명주요 기능
중개인심사·승인된 검증 중개자 (내부 이용자를 풀로 보유)프로필 심사, 인맥 풀 관리, 매칭 주선(풀 내/교차), 1:1 채팅, 수익 정산
내부 이용자특정 중개인 풀에 소속된 이용자프로필 관리, 일일 추천 탐색, 관심 표현, 중개인과 채팅, 매칭 진행·결제
외부 이용자아직 중개인에 소속되지 않은 이용자제한된 탐색, 내부 이용자에게 매칭 신청, 중개인 풀 가입 신청
관리자운영팀 (PC 웹)대시보드, 회원·중개인 승인, 매칭 모니터링, 정산 승인, 콘텐츠 관리

3. 기능 요구사항

3.1 공통 및 사용자 기능 (내부/외부 이용자)

  • [필수] 역할(이용자/중개인)에 따른 가입 절차 및 로그인 (Apple/Google/Kakao SSO)
  • [필수] 프로필 작성·관리 (사진 비공개 원칙, 가치관·직업 등 상세 정보)
  • [필수] 탐색(둘러보기): 매일 소수 이성 프로필(사진 제외) 추천, '관심' 표현
  • [필수] 관심함: 관심 표현 이성 목록 관리
  • [필수] 매칭 진행도 관리: 제안 확인, 수락/거절, 단계별 현황 조회
  • [필수] 중개인과 1:1 채팅 (이용자 간 채팅 차단)
  • [필수] 코인 충전 및 매칭 최종 수락 시 결제 (인앱결제 + 웹결제)
  • [필수] 외부 이용자의 중개인 풀 가입 신청 기능
  • [필수] 라운지: 운영자 제공 설문·심리테스트 콘텐츠 이용
  • [선택] 라운지: 신뢰 등급 높은 중개인 주최 단체 미팅 참여

3.2 중개인 기능

  • [필수] 중개인 프로필 등록 및 관리자 승인 절차
  • [필수] 내 인맥 풀 관리 (소속 내부 이용자 목록 조회·관리)
  • [필수] 전체 이용자(내부/외부) 프로필 탐색
  • [필수] 매칭 주선: 본인 풀 내, 다른 중개인 풀 교차 매칭 제안
  • [필수] 이용자로부터 받은 매칭 요청 수락/거절
  • [필수] 중개 히스토리 및 수익 정산 내역 조회
  • [필수] 다수 이용자 동시 1:1 채팅 관리

3.3 관리자 시스템 (웹)

  • [필수] 대시보드: 회원 현황, 매칭 현황, 매출 등 핵심 지표
  • [필수] 회원 관리: 전체 회원 조회·제재, 중개인 승인 관리
  • [필수] 매칭 관리: 전체 매칭 진행 모니터링
  • [필수] 정산 관리: 중개인 수익금 정산 및 이력 관리
  • [필수] 콘텐츠 관리: 라운지 설문·심리테스트 등록·관리

4. 비기능 요구사항

항목요구사항
보안개인정보(연락처·사진) 최종 매칭 전까지 비공개, 필드 레벨 암호화(AES-256), 채팅 메시지 암호화, 미디어 Signed URL(TTL)
성능실시간 채팅·매칭 요청 지연 없는 처리 (Socket.IO + Redis Pub/Sub, p95 응답 300ms 이하 목표)
가용성AWS 멀티 AZ 배포, 장애 시 자동 복구 구조
확장성사용자 증가에 따른 수평 확장 가능한 스테이트리스 API 구조
접근성iOS/Android 최신 2개 메이저 버전 지원
컴플라이언스개인정보보호법·위치정보법 등 국내 규정 준수, NDA 체결 후 진행

5. 기술 제약사항

  • 모바일: iOS(StoreKit 2), Android(Google Play Billing Library v6+) 정책 준수
  • 결제: 앱스토어 정책상 디지털 재화(코인)는 인앱결제 필수, 실물 서비스는 웹결제 활용
  • 실시간: 채팅·매칭 알림은 WebSocket 우선, Fallback으로 푸시 알림(FCM/APNS)
  • 데이터 로컬라이즈: 국내 법령 준수 위해 AWS 서울 리전 사용
  • 본인인증: NICE 본인인증 또는 이동통신사 PASS 연동 (상세는 착수 후 협의)

6. 범위 외 사항 (Out of Scope)

  • 상세 사업 정책(수수료율·분배 비율) 수립 자문 — 클라이언트 제공 기준 반영
  • 마케팅 운영·CRM 캠페인 기능 (Phase 2 고도화 대상)
  • 이용자 간 직접 채팅·영상통화 (요구사항상 차단)
  • AI 기반 매칭 추천 고도화 (MVP 이후 단계적 도입)
  • 정기 결제·구독 모델 (Phase 2)
  • 다국어 지원 (MVP는 한국어만)

디지털 명함 & 근거리 네트워킹 매칭 플랫폼

B2B 네트워킹 · 2025.05~2025.08 (3개월)

프로젝트 개요

BLE 근거리 탐색 기반 디지털 명함 교환과 E2E 암호화 메시징, OCR 명함 스캔, 외부 슈퍼앱(Zalo) 연동까지 수행한 매칭·네트워킹 플랫폼을 3개월 MVP로 런칭한 프로젝트입니다.

핵심 기능

  • 사용자 매칭 및 프로필 탐색 (BLE 근거리 인식)
  • Signal Protocol 기반 E2E 암호화 1:1 메시징
  • OCR 명함 스캔, 슈퍼앱(Zalo) OAuth 연동
  • 12 마이크로서비스, 63 API 엔드포인트
  • Flutter 크로스플랫폼 (iOS/Android)

본 프로젝트와의 유사점

  • 매칭 플랫폼 핵심 도메인: 프로필 탐색 → 관심 표현 → 매칭 → 채팅 플로우가 본 프로젝트와 동일한 패턴
  • Flutter 크로스플랫폼: iOS/Android 단일 코드베이스 런칭 경험
  • E2E 암호화 메시징: 본 프로젝트의 채팅 암호화 요건을 직접 경험한 구조 이식 가능
  • 3개월 MVP 딜리버리: 동일한 일정·규모의 프로젝트를 완주한 경험

기술 스택

복잡한 역할·권한 SaaS

전자결재 SaaS (다단계 워크플로우 + 7 역할 RBAC)

B2B SaaS · 2026.01~2026.03 (약 9주)

프로젝트 개요

8종의 결재 액션과 다단계 승인 워크플로우, 7가지 사용자 역할과 6계층 보안 체계를 갖춘 전자결재 SaaS. 관리자 대시보드·외부 셀프서비스 포탈·실시간 공동편집까지 단일 플랫폼에 통합한 기업용 시스템입니다.

핵심 기능

  • 다단계 전자결재 엔진 (제안→검토→승인→반려→재상신 상태 머신)
  • 7 사용자 역할 × 6계층 보안 (CASL 기반 RBAC)
  • 관리자 대시보드(회원·결재·정산·모니터링)
  • 토큰 기반 외부 셀프서비스 포탈 (제한된 탐색·신청 플로우)
  • PR 504건, 50+ 페이지, 120-150 API, 34 엔티티

본 프로젝트와의 유사점

  • 복잡한 역할·권한 관리: 중개인/내부/외부/관리자 4개 역할과 매칭·정산 상태 머신을 동일한 패턴으로 이식
  • 관리자 대시보드: 회원 관리·승인 관리·정산 관리가 본 프로젝트 관리자 웹과 구조상 동일
  • 승인 워크플로우 엔진: 중개인 승인·매칭 제안·수락/거절 플로우를 상태 머신으로 설계한 경험 직접 활용
  • 외부→내부 진입 플로우: 외부 이용자의 중개인 풀 가입 신청이 셀프서비스 포탈 구조와 유사

프로젝트 개요

시니어 주간보호 기관을 위한 멀티테넌트 관리 플랫폼으로, Flutter 모바일 앱과 Next.js 관리자 웹, NestJS 백엔드, 6 플랫폼 지원, 실시간 알림과 전자서명·AI 분석까지 수행한 대규모 헬스케어 SaaS입니다.

핵심 기능

  • Flutter 앱 + Next.js 관리자 웹 + NestJS 백엔드 통합 플랫폼
  • 멀티테넌트 아키텍처 (기관별 데이터 격리)
  • 실시간 알림, 보호자 커뮤니케이션, 전자서명
  • AWS CDK 기반 IaC 배포, 140+ API, 133K+ LOC
  • AI 건강 분석 (OpenAI 연동)

본 프로젝트와의 유사점

  • 전체 아키텍처 일치: Flutter 앱(이용자/중개인) + Next.js 관리자 웹 + NestJS 백엔드 = 본 프로젝트와 동일 구조
  • 실시간 통신: 채팅·푸시 알림 구조를 동일한 패턴으로 재사용
  • 관리자 웹 승인·모니터링: 회원·콘텐츠·정산 관리 패턴 이식 가능
  • AWS 운영 배포: IaC 기반 안정적 배포·모니터링 경험

기술 스택

Flutter NestJS Next.js TypeScript MySQL AWS CDK Docker
